Detection Engine

Three independent layers, one verdict

Each layer fails differently — so they're stacked in depth. A package must survive reputation, static inspection, and live execution to come back clean.

LAYER 01

Reputation

Surface · metadata & intelligence
  • Typosquatting via Levenshtein distance
  • Suspicious age, low release count, hidden author
  • Maintainer e-mail change (account takeover)
  • Known CVEs / GHSAs via OSV
LAYER 02

Static

Mid-water · code without execution
  • Dangerous AST patterns, hook-context aware
  • YARA rules: Base64 eval, reverse shells, tokens
  • Shannon entropy → obfuscated payloads
  • Intra-procedural taint analysis (source → sink)
LAYER 03

Dynamic

Deep · isolated execution
  • Hardened Docker container, --network none
  • strace syscall tracing: connect / execve / open
  • Python function instrumentation
  • Flags egress, file leaks, suspicious spawns

Weighted & additive, with cross-layer correlation bonuses (obfuscation and outbound network scores more than either alone) and a strict whitelist that suppresses false positives on trusted packages.

Architecture

A modular pipeline, decoupled by a data contract

The core orchestrates three scanners that all emit the same JSON shape. The scorer, dashboard and CI gate consume any module interchangeably — which let four people build in parallel.

Positioning

Why DepthCharge

Getting Started

Up and running in minutes

Python and an optional Docker daemon for the sandbox. No daemon? DepthCharge degrades gracefully and runs reputation + static.

01

Clone the repo

git clone https://github.com/MAITO465/depthcharge.git
02

Install dependencies

cd depthcharge && pip install -r requirements.txt
03

Launch the dashboard

python modules/dashboard/app.py # http://localhost:5001
04

Or scan from the CLI

python depthcharge.py scan requests
